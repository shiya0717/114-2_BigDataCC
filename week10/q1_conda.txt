=== 任務 6：觀念回答 ===
Q1：為什麼不要直接在 base 環境安裝專案套件？這樣做有什麼壞處？
A1：避免版本衝突。base 是管理核心，若所有專案都裝在一起，會導致套件互相干擾、Conda 變慢甚至損壞。獨立環境能讓不同專案擁有各自專屬的版本。

Q2：conda install 和 pip install 有什麼差異？什麼情況下會建議用 pip？
A2：conda 像全能管家，能管 Python 以外的系統庫（如 C++、CUDA）；pip 則是 Python 專科。當 Conda 找不到特定套件時，才建議使用 pip 補足。

Q3：environment.yml 的用途是什麼？換電腦時要怎麼用它快速重建環境？
A3：它是環境的配方表，記錄所有套件版本以確保他人能完全複製你的環境。

重建指令：conda env create -f environment.yml
